# Releasing

This document describes how to create a new release of `n8n-nodes-invoice-api-xhub`.

## Quick Start

```bash
pnpm release          # Patch release (1.0.7 → 1.0.8)
pnpm release:minor    # Minor release (1.0.7 → 1.1.0)
pnpm release:major    # Major release (1.0.7 → 2.0.0)
```

That's it. The script handles everything from testing to npm publishing.

## What Happens

The release script (`scripts/release.sh`) performs these steps in order:

1. **Validates** — checks you're on `main`, working directory is clean, `gh` CLI is authenticated
2. **Pulls latest** — `git pull origin main`
3. **Quality checks** — runs `pnpm build`, `pnpm lint`, `pnpm test`
4. **Bumps version** — updates `version` in `package.json`
5. **Commits and tags** — `chore: release v{version}` commit + `v{version}` git tag
6. **Pushes** — pushes commit and tag to GitHub
7. **Creates GitHub Release** — via `gh release create` with auto-generated release notes
8. **npm publish** — triggered automatically by the `publish.yml` GitHub Actions workflow

## Prerequisites

- **Git**: clean working directory, on `main` branch
- **GitHub CLI** (`gh`): installed and authenticated (`gh auth login`)
- **pnpm**: installed
- **Node.js**: >= 22.0
- **npm token**: configured as `NPM_TOKEN` secret in the GitHub repository settings

## Version Bumping

Follow [Semantic Versioning](https://semver.org/):

| Bump | When | Example |
|------|------|---------|
| `patch` | Bug fixes, dependency updates, documentation changes | 1.0.7 → 1.0.8 |
| `minor` | New features, new operations, new format/country support | 1.0.7 → 1.1.0 |
| `major` | Breaking changes to node parameters or behavior | 1.0.7 → 2.0.0 |

## Release Notes

Release notes are auto-generated by GitHub based on commits since the last release. To get useful release notes, write descriptive commit messages:

```
feat: add Parse Auto-Detect operation
fix: handle empty binary data in parse operation
docs: update API reference for v1.1.0
```

GitHub groups commits by contributor and links to PRs automatically.

```
           ▼
┌─────────────────────────────┐
│  .github/workflows/         │

           │

```

## Troubleshooting

### "Must be on 'main' branch"

Switch to main and merge your changes first:

```bash
git checkout main
git merge your-feature-branch
```

### "Working directory is not clean"

Commit or stash your changes:

```bash
git stash          # or
git add -A && git commit -m "your message"
```

### "GitHub CLI not authenticated"

```bash
gh auth login
```

### npm publish fails in CI

Check the `NPM_TOKEN` secret in **GitHub → Repository Settings → Secrets and variables → Actions**. The token needs `publish` permissions on npmjs.com.

### Tests fail during release

Fix the failing tests first. The release script intentionally aborts if any quality check fails.
